## Сделать вмешательство в жизненный цикл бина
Вмешательство в жизненный цикл сделано в нескольких вариациях:
1) Через аннотацию @PostConstruct из пакета jakarta, в классе
   [HomeUIController](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/controller/home/HomeUIController.java)
2) Напрямую через имплементацию интерфейса **InitializingBean** и реализацию его метода  **afterPropertiesSet**
   , в классе
   [HomeUIController](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/controller/home/HomeUIController.java)

В рамках этого вмешательства для удобства демонстрации просто происходят дополнительные записи в лог.

## Использовать аспекты
В качестве примера аспектов написан класс со срезами(pointcut) 
[AdviceExecutor](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/aspect/logging/AdviceExecutor.java)
и класс с советами(advice) [PointCutSearcher](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/aspect/logging/PointCutSearcher.java)

Так же для эксперимента написан аспект который использует аннотации 
[EnableLoggingr](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/aspect/annotaion/EnableLogging.java)
как срез   и где pointcut  и advice объединены вместо из-за этого подхода.

Примеры использования:
* обычный аспект используется для логирования контроллеров в **home**, **profile** и в контроллерах папки безопасности.
* метод через аннотацию применен к методу в классе
[QuestsUIController](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/controller/quests/QuestsUIController.java)

## Spring Data

### JDBC Template
С помощью этого подхода сделан репозиторий
[JdbcTemplateRoleRepository](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/repository/jdbc_template/JdbcTemplateRoleRepository.java).

Так же есть вспомогательный класс с запросами
[RoleQueries](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/repository/jdbc_template/RoleQueries.java)
и класс-мапер
[RoleRowMapper](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/mapper/RoleRowMapper.java)
для преобразования из ResultSet  в соотвествующий класс Role.

Примечание: так как Role связан с User в ORM модели и нужен для переопределенных методов работы с security, у него есть и репозиторий через интерфейсы.
### Работа с репозиториями через описание интерфейсов

Пример, где применяется подход **Spring Derivation** можно посмотреть в классе
[PostRepository](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/repository/PostRepository.java).

Пример, где применяется переопределение метода через **JPQL**  и применение **fetch**, можно посмотреть в классе
[StageRepository](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/repository/StageRepository.java).

Пример использования транзакций, можно увидеть в сервисе
[QuestService](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/service/QuestService.java).

## Spring MVC
Используется Thymeleaf в качестве шаблонизатора.

Так же на примере DTO
[AuthRequest](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/security/dto/AuthRequest.java)
 используется @Valid проверки.

Дополнительно сделаны кастомные исключения связанны с этими проверками в файле
[AuthControllerExceptionHandler](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/security/controller/AuthControllerExceptionHandler.java)

Примечание: после изменение работы авторизации через jwt, ошибку можно увидеть только либо в логах, либо в консоли браузера при отладке.

## Тестирование

### Unit-тестирование
Тестами покрыт класс
[QuestService](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/service/QuestService.java).

Это реализовано в файле
[QuestServiceTest](src/test/java/com/javarush/hibernate_final/ostapenko/hibernate/service/QuestServiceTest.java).

В ходе тестирования используется кастомный assert -
[QuestAssert](src/test/java/com/javarush/hibernate_final/ostapenko/hibernate/custom/QuestAssert.java).

### Тестирование для контроллера
В качестве примера выступает контроллер для авторизации
[AuthControllerTest](src/test/java/com/javarush/hibernate_final/ostapenko/hibernate/security/controllers/AuthControllerTest.java)

При его тестировании используем только бин контроллера
[AuthController](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/security/controller/AuthController.java).

Остальные объекты реализованы с помощью Mock-ов.

## Интернационализация
Для удобства работы с Resource Bundle потребуется установить плагин **Resource Bundle Editor**.

Для работы интернационализации создан дополнительный конфигурационный класс
[MessageConfig](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/config/MessageConfig.java).

Ресурсы для работы находятся в папке 
[language](src/main/resources/language/).

Интернационализация применена в файле слоев [header](src/main/resources/templates/layout/header.html).

### Spring Security

Необходимые для работы файл находятся в папке 
[security](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/security/).

Сюда вынесены:
* контроллер отвечающий за авторизацию и связанны с ним DTO
* в папке
[jwt](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/security/jwt)
вынесены сервис, фильтр и настройки диалекта(для взаимодействия с таймлиф) при работе с jwt.

* и [CustomUserDetailService](src/main/java/com/javarush/hibernate_final/ostapenko/hibernate/security/service/CustomUserDetailService.java)
нужен, чтобы считывались данные пользователей из нашей базы
